# LeetCode 动态规划路径类总结

`不同路径` 和 `最小路径和` 都属于典型的路径类 DP。

这类题的核心在于：

- 题目通常描述为从起点走到终点
- 每个位置的答案依赖于它前面的某些位置
- 本质是根据当前位置的前驱状态，推导当前位置的结果

两道题虽然题目问法不同，但分析框架基本一致，区别主要在于 `dp[i][j]` 的定义，以及状态转移时是求和、取最值，还是判断是否可达。

## 一、什么是路径类 DP

路径类 DP 一般出现在网格、矩阵、坐标移动这类题目中。

常见特征：

- 有明确的起点和终点
- 移动方向通常受限，例如只能向右、向下
- 需要求方案数、最小代价、最大收益或是否可达

这类题的本质可以概括为：

> 以当前位置为终点，思考它可以由哪些前驱位置转移而来。

## 二、通用分析步骤

遇到路径类 DP，可以按照下面的顺序思考。

### 1. 抽象问题模型

先把题目抽象成一个网格路径问题，明确：

- 起点是谁
- 终点是谁
- 每一步允许怎么走
- 某些格子是否有额外代价或限制

### 2. 定义状态

定义 `dp[i][j]` 表示什么，这是整道题最关键的一步。

路径类 DP 中，常见的状态含义有三类：

- `方案数`：走到 `(i, j)` 有多少种方法
- `最优值`：走到 `(i, j)` 的最小代价或最大收益
- `可达性`：能否走到 `(i, j)`

对应到两道题：

- `不同路径`：`dp[i][j]` 表示从起点走到 `(i, j)` 的路径数量
- `最小路径和`：`dp[i][j]` 表示从起点走到 `(i, j)` 的最小路径和

### 3. 找状态转移来源

思考到达当前位置 `(i, j)` 的最后一步从哪里来。

如果题目规定只能向右或向下走，那么到 `(i, j)` 只可能来自：

- 上边 `(i - 1, j)`
- 左边 `(i, j - 1)`

这一步是路径类 DP 的核心。不要只想“从起点怎么走到终点”，而要想“终点的最后一步从哪来”。

### 4. 写状态转移方程

来源确定之后，状态转移方程通常就能直接写出来。

#### 不同路径

到 `(i, j)` 的路径数等于从上边来的路径数加上从左边来的路径数：

```text
dp[i][j] = dp[i - 1][j] + dp[i][j - 1]
```

#### 最小路径和

到 `(i, j)` 的最小路径和，等于上边和左边较小的那个，再加上当前格子的值：

```text
dp[i][j] = min(dp[i - 1][j], dp[i][j - 1]) + grid[i][j]
```

这说明路径类 DP 的共同框架是一样的，只是“合并前驱状态”的方式不同：

- 求方案数，通常用 `+`
- 求最值，通常用 `min` 或 `max`
- 求是否可达，通常用逻辑判断

### 5. 处理初始化

初始化是路径类 DP 中最容易出错的地方，尤其要注意：

- 起点
- 第一行
- 第一列

因为第一行只能从左边过来，第一列只能从上边过来。

#### 不同路径的初始化

起点只有一种走法：

```text
dp[0][0] = 1
```

第一行每个位置都只能从左边走过来，所以都是 `1`。

第一列每个位置都只能从上边走过来，所以也都是 `1`。

#### 最小路径和的初始化

起点的最小路径和就是它自己：

```text
dp[0][0] = grid[0][0]
```

第一行只能一直从左边累加：

```text
dp[0][j] = dp[0][j - 1] + grid[0][j]
```

第一列只能一直从上边累加：

```text
dp[i][0] = dp[i - 1][0] + grid[i][0]
```

可以总结为：

- `方案数题` 的边界通常是固定值
- `代价题` 的边界通常是前缀累加

### 6. 确定遍历顺序

如果 `dp[i][j]` 依赖上方和左方，那么遍历顺序通常是：

- 从上到下
- 从左到右

即：

```text
for i in [0..m-1]:
    for j in [0..n-1]:
```

原则是：

> 当前状态依赖谁，就要先把谁算出来。

## 三、两道题的统一模板

从更高层去看，`不同路径` 和 `最小路径和` 都可以归纳成下面的形式：

```text
dp[i][j] = combine(dp[i - 1][j], dp[i][j - 1], 当前格子信息)
```

区别只在于 `combine` 的定义不同：

- `不同路径`：`combine = 上 + 左`
- `最小路径和`：`combine = min(上, 左) + 当前值`

因此，遇到新的路径类 DP 题时，可以先问自己：

- `dp[i][j]` 表示方案数、最值，还是可达性
- `(i, j)` 能由哪些方向转移而来
- 这些来源之间是做加法、取最值，还是做逻辑判断

## 四、路径类 DP 的常见分类

### 1. 计数型

典型题目：`不同路径`

特征：

- 问有多少种方案
- 转移通常是把所有前驱方案数相加

### 2. 最值型

典型题目：`最小路径和`

特征：

- 问最小代价或最大收益
- 转移通常是在前驱中取 `min` 或 `max`

### 3. 可达型

特征：

- 问某个位置能不能到
- 转移通常是判断某个前驱是否可达

### 4. 带障碍型

特征：

- 某些格子不能走
- 通常这些格子对应的状态直接置为 `0` 或不可达

### 5. 带权路径型

特征：

- 每个格子带有代价或收益
- 转移时需要把当前格子的权值加进去

### 6. 多方向转移型

特征：

- 不止可以从上、左转移
- 可能还允许斜着走，或者从多个方向移动

本质仍然不变，只是前驱状态变多了。

## 五、做题时的固定提问模板

以后遇到路径类 DP，可以固定问自己下面几个问题：

1. `dp[i][j]` 表示什么
2. 到达 `(i, j)` 的最后一步从哪里来
3. 多个来源之间如何合并
4. 起点如何初始化
5. 第一行、第一列如何初始化
6. 遍历顺序如何保证前驱已计算完成
7. 是否可以进行空间优化

只要这几个问题回答清楚，路径类 DP 基本就能写出来。

## 六、空间优化思路

如果 `dp[i][j]` 只依赖：

- 上一行同一列
- 当前行前一列

那么二维数组通常可以压缩成一维数组。

例如用 `dp[j]` 表示当前处理到这一行时，第 `j` 列的答案：

- 更新前的 `dp[j]`，表示上一行的值
- 更新后的 `dp[j - 1]`，表示当前行左边的值

因此，很多网格路径题都可以把空间复杂度从 `O(mn)` 优化到 `O(n)`。

不过在实际做题时，建议：

- 先写二维 DP，把状态和转移理顺
- 再考虑是否进行滚动数组优化

## 七、为什么路径类问题适合用 DP

因为它同时满足动态规划的两个核心条件：

### 1. 最优子结构

当前位置的答案，可以由前面位置的答案推导出来。

### 2. 重叠子问题

多个位置在计算时，会重复用到相同的前驱状态。

例如不同位置都可能依赖同一个上方或左方位置的结果，所以使用 DP 可以避免重复计算。

## 八、一句话总结

路径类 DP 的本质就是：

> 以当前位置为终点，思考它由哪些前驱状态转移而来。

做题时按下面的顺序分析即可：

1. 定义 `dp[i][j]` 的含义
2. 找当前位置的前驱状态
3. 写状态转移方程
4. 处理起点、第一行、第一列的初始化
5. 确定正确的遍历顺序
6. 最后考虑是否进行空间优化

## 九、通用模板

```text
1. 定义状态：
   dp[i][j] = 从起点到 (i, j) 的某种结果

2. 找转移来源：
   通常来自上、左，也可能来自更多方向

3. 写转移方程：
   计数：dp[i][j] = ...
   最值：dp[i][j] = min/max(...)
   可达：dp[i][j] = true/false

4. 初始化：
   处理起点、第一行、第一列

5. 确定遍历顺序：
   保证前驱状态先于当前状态被计算
```
