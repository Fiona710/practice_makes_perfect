# 广度优先遍历学习思路

## 对话背景

### 用户问题一

> Java程序员跳槽找工作，现在在刷LeetCode上的题目，对深度优先遍历算法不了解，请给出学习广度优先遍历算法的辅导建议。

### 回复内容一

## 先抓核心

- 广度优先遍历 `BFS` 的本质是：`一层一层地找`，离起点近的先处理，离起点远的后处理。
- 如果你现在对深度优先遍历 `DFS` 还不熟，不影响先学 `BFS`，因为两者思路差别很大。
- 你可以把它们这样区分：
  - `DFS`：一条路走到底，再回退。
  - `BFS`：先把当前层都看完，再进入下一层。

## 适合 Java 程序员的理解方式

- 把 `BFS` 想成“任务队列”最容易。
- 需要一个 `队列 Queue`：
  - 先把起点放进去。
  - 每次从队头拿一个点出来处理。
  - 再把它相邻、还没访问过的点放到队尾。
- 这样天然就保证了“按层遍历”。

## 先学这 4 个点

- 学会什么时候用 `Queue`
- 学会怎么写 `visited` 去重
- 学会“按层遍历”的写法
- 学会 `BFS` 为什么常用于“最短步数”

## Java 标准模板

```java
import java.util.*;

public class BFSDemo {
    public void bfs(int start, List<List<Integer>> graph) {
        Queue<Integer> queue = new LinkedList<>();
        boolean[] visited = new boolean[graph.size()];

        queue.offer(start);
        visited[start] = true;

        while (!queue.isEmpty()) {
            int cur = queue.poll();
            System.out.println(cur);

            for (int next : graph.get(cur)) {
                if (!visited[next]) {
                    visited[next] = true;
                    queue.offer(next);
                }
            }
        }
    }
}
```

## 这个模板你重点看什么

- `queue.offer(start)`：起点入队
- `queue.poll()`：取出当前节点
- `if (!visited[next])`：防止重复访问
- `visited[next] = true` 要在 `入队时` 做，不要等出队再做

## 为什么 LeetCode 常考 BFS

- 求“最少步数”
- 求“从起点到终点最短路径”
- 树的层序遍历
- 矩阵中的扩散问题
- 图中的连通块、状态转换

- 因为 `BFS` 按层扩展，所以第一次到达目标时，通常就是最短距离。

## 建议你的学习顺序

- 第一步：先学二叉树层序遍历
- 第二步：再学网格图 `matrix BFS`
- 第三步：再学无权图最短路径
- 第四步：最后学状态压缩、双向 BFS 这种进阶题

## 推荐刷题路线

- 简单入门：
  - 二叉树的层序遍历
  - 二叉树的右视图
  - 二叉树每层平均值
- 矩阵 BFS：
  - 岛屿数量
  - 腐烂的橘子
  - `01` 矩阵
- 图 BFS：
  - 钥匙和房间
  - 单词接龙
  - 打开转盘锁

## 学习时最容易犯的错

- `visited` 标记太晚，导致重复入队
- 只会普通遍历，不会“分层统计步数”
- 把树题和图题混了，图题通常必须去重
- 队列里只存节点，不知道何时记录层数

## 层数写法模板

```java
public int bfsLevel(int start, int target, List<List<Integer>> graph) {
    Queue<Integer> queue = new LinkedList<>();
    boolean[] visited = new boolean[graph.size()];

    queue.offer(start);
    visited[start] = true;

    int step = 0;

    while (!queue.isEmpty()) {
        int size = queue.size();

        for (int i = 0; i < size; i++) {
            int cur = queue.poll();
            if (cur == target) {
                return step;
            }

            for (int next : graph.get(cur)) {
                if (!visited[next]) {
                    visited[next] = true;
                    queue.offer(next);
                }
            }
        }
        step++;
    }

    return -1;
}
```

## 你可以这样记忆

- `队列` 决定顺序
- `visited` 防止重复
- `size` 控制一层
- `step` 记录距离

## 给你的辅导建议

- 不要一开始就背模板，先拿纸画一棵树，模拟队列变化
- 每做一道题，都先问自己一句：`这是按层扩展吗？这是求最短步数吗？`
- 每次写 BFS 都固定套这 4 步：
  - 初始化队列
  - 标记起点已访问
  - 循环出队
  - 遍历邻居并入队
- 同一题目再对比写一次 `DFS`，你会更快理解两者区别

## 如果你想高效准备跳槽

- 面试中 `BFS` 高频度很高，尤其是：
  - 二叉树层序遍历
  - 矩阵最短路径
  - 图最短步数
- 你的目标不是“把 BFS 讲得很学术”，而是做到：
  - 一眼识别题型
  - 5 分钟写出模板
  - 能说明时间复杂度和为什么用 BFS

## 最后给你一个判断口诀

- `一层一层找，用 BFS`
- `最短步数，优先想 BFS`
- `走到底再回头，是 DFS`

## 广度优先遍历算法，为什么能解决最短路径问题

广度优先遍历（BFS）之所以能解决 最短路径 问题，核心原因是：

- 它是按“层”扩展的
- 第 k 层表示“从起点出发，刚好走了 k 条边能到达的所有点”
- 所以它第一次到达某个点时，使用的一定是 边数最少 的路径
直观理解

把起点看成第 0 层：

- 第 0 层：起点自己
- 第 1 层：一步能到的点
- 第 2 层：两步能到的点
- 第 3 层：三步能到的点
BFS 用队列保证：

- 先处理完所有“1 步可达”的点
- 再处理所有“2 步可达”的点
- 再处理所有“3 步可达”的点
因此，当某个点第一次被访问到时，不可能还存在一条更短的路径没被发现，因为更短的路径所在的层早就应该被处理过了。

为什么“第一次到达”就是最短

假设 BFS 第一次到达节点 v 时用了 4 步。

如果其实存在一条只要 3 步的路径到 v ，那么：

- v 应该在第 3 层就被发现
- BFS 会先搜索第 0、1、2、3 层
- 不可能等到第 4 层才第一次看到它
这就矛盾了。

所以，第一次到达 v 的路径，一定是最短路径。

适用前提

这里的“最短路径”指的是：

- 无权图 中的最短路径
- 或者 每条边权重都相同 时的最短路径
