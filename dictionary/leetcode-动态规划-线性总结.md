# LeetCode 动态规划-线性总结

## 一、什么是线性动态规划

### 1. 什么是线性 dp

- `线性动态规划`，通常简称 `线性 dp`，指的是状态按照某种线性顺序推进，当前状态由前面若干个状态转移而来的一类动态规划。
- 这里的“线性”不是说状态一定只是一维，而是说问题的推进顺序是线性的，通常沿着：
  - 数组下标
  - 字符串位置
  - 天数
  - 台阶编号
  - 时间顺序
- 这类题最常见的状态形式是：

```text
dp[i]
```

- 也可能是：

```text
dp[i][state]
```

- 它们都表示：`处理到第 i 个位置时` 的某种答案。

### 2. 为什么它叫“线性”动态规划

- 因为这类题的子问题天然可以按顺序排列。
- 比如：
  - 前 `i` 个元素的最优值
  - 以第 `i` 个元素结尾的最优值
  - 第 `i` 天结束时处于某种状态的最优值
- 当前问题只依赖前面已经计算过的状态，因此可以从左到右逐步递推。

### 3. 线性 dp 的核心思想

- 把原问题拆成很多“处理到某个位置时”的子问题。
- 用更短前缀或更早位置的答案，推出当前位置的答案。
- 本质上是在做一句话：

```text
当前位置的答案，由前面位置的答案递推出来
```

- 所以线性 dp 的关键不只是会写状态，还要想清楚：
  - `dp[i]` 到底表示前缀最优，还是以 `i` 结尾
  - 当前状态依赖哪些更早状态
  - 遍历顺序怎样保证依赖已经算出来

### 4. 线性 dp 和其他 dp 的区别

- 一维前缀型 `dp`、结尾型 `dp`、带有限状态的一维 `dp`，都属于线性 dp 的范围。
- 它和其他常见动态规划的区别可以粗略理解成：
  - `线性 dp`：沿位置顺序推进
  - `区间 dp`：围绕区间 `[i, j]` 展开
  - `双序列 dp`：围绕两个序列的前缀展开
  - `背包 dp`：围绕容量和物品选择展开
  - `树形 dp`：围绕树结构展开

### 5. 线性 dp 最常见的状态定义方式

- 前缀型：

```text
dp[i] = 前 i 个元素的最优值 / 方案数 / 最少代价
```

- 结尾型：

```text
dp[i] = 以第 i 个元素结尾的最优值
```

- 双状态或多状态型：

```text
dp[i][0/1]
dp[i][k]
```

- 它们表示：处理到第 `i` 个位置时，处于不同状态下的答案。

### 6. 线性 dp 的思考顺序

- 第一步：定义 `dp[i]` 或 `dp[i][state]` 表示什么
- 第二步：思考第 `i` 个位置的决策是什么
- 第三步：确定从哪些旧状态转移而来
- 第四步：初始化边界状态
- 第五步：根据依赖关系确定遍历顺序
- 第六步：确定最终答案在什么位置

- 可以把它总结成一句口令：

```text
先定位置状态，再想当前位置如何由前面位置推出
```

## 二、什么场景下需要使用线性 dp

### 1. 线性 dp 主要解决什么问题

- 线性 dp 主要解决的是：`和顺序有关，并且当前决策依赖前面历史结果` 的最优值、最少代价、最长长度、方案数问题。
- 题目表面不同，但本质上常见有下面几类：
  - 处理到某个位置时的最优结果
  - 以某个位置结尾的最优结果
  - 从前面选择若干个位置组成最优结构
  - 统计到某个位置为止的方案数
  - 在有限状态下沿时间或位置推进

### 2. 什么时候优先想到线性 dp

- 当题目给的是一个数组、字符串、天数、台阶、位置序列时，要优先考虑线性 dp。
- 特别是题目里出现这些词时，要提高敏感度：
  - 前 `i` 个
  - 到第 `i` 个位置
  - 以 `i` 结尾
  - 第 `i` 天
  - 最长
  - 最大
  - 最小
  - 方案数
  - 选或不选

### 3. 典型识别特征

- `特征一：问题具有明显顺序`
  - 比如数组下标、字符位置、时间推进、楼梯层数。

- `特征二：当前状态只依赖前面状态`
  - 比如 `dp[i - 1]`、`dp[i - 2]` 或某个 `dp[j] (j < i)`。

- `特征三：可以定义“处理到第 i 个位置”的子问题`
  - 如果题目能自然写成“前 `i` 个元素的答案”或“以 `i` 结尾的答案”，通常就很像线性 dp。

- `特征四：贪心不容易直接证明正确`
  - 局部最优的选择可能影响后续，因此需要保留历史最优信息。

- `特征五：存在重复子问题`
  - 暴力搜索会重复计算很多前缀状态或结尾状态。

### 4. 线性 dp 最常见的几类问题

#### 1. 简单递推类

- 当前状态只依赖前面少量状态。
- 常见形式：

```text
dp[i] <- dp[i - 1], dp[i - 2]
```

- 代表场景：
  - 爬楼梯
  - 最小花费爬楼梯
  - 打家劫舍

#### 2. 以 i 结尾的最优值类

- 当前状态强调“必须以当前位置结尾”。
- 常见形式：

```text
dp[i] = 以 i 结尾的最优值
```

- 代表场景：
  - 最大子数组和
  - 最长有效括号
  - 乘积最大子数组

#### 3. 枚举前驱类

- 当前状态需要从所有前驱位置中找能转移过来的最优解。
- 常见形式：

```text
dp[i] = max/min(dp[j] + value), j < i
```

- 代表场景：
  - 最长上升子序列
  - 最大整除子集
  - 最长数对链

#### 4. 多状态状态机类

- 每个位置有多个状态，例如持有、不持有、删除过、未删除等。
- 常见形式：

```text
dp[i][state]
```

- 代表场景：
  - 股票系列
  - 删除一次得到最大子数组和
  - 带限制的选择问题

#### 5. 前缀划分和方案数类

- 当前答案由“最后一步”或“最后一段”决定。
- 常见形式：

```text
dp[i] = sum/max/min(dp[j] + cost(j, i))
```

- 代表场景：
  - 解码方法
  - 单词拆分
  - 完全平方数

## 三、线性 dp 的解题思路和模板

### 1. 做题时的通用思考流程

#### 第一步：确定阶段

- 通常阶段就是位置 `i`。
- 例如：
  - 第 `i` 个元素
  - 前 `i` 个字符
  - 第 `i` 天
  - 第 `i` 级台阶

#### 第二步：定义状态

- 先明确一句话：

```text
dp[i] / dp[i][state] 表示什么
```

- 常见定义方式：
  - `dp[i] = 前 i 个元素的最优值`
  - `dp[i] = 以 i 结尾的最优值`
  - `dp[i] = 到位置 i 为止的方案数`
  - `dp[i][0/1] = 处理到 i 时处于不同状态的最优值`

#### 第三步：分析最后一步或当前位置决策

- 线性 dp 最核心的问题通常是：
  - 第 `i` 个位置选还是不选
  - 是接在前面后面，还是从当前位置重新开始
  - 从哪个前驱 `j` 转移过来
  - 当前状态是否需要切换

#### 第四步：写出状态转移

- 常见转移方式有三种：

- 少量前驱转移：

```text
dp[i] = f(dp[i - 1], dp[i - 2], 当前信息)
```

- 枚举前驱转移：

```text
dp[i] = max/min(dp[j] + value), j < i
```

- 多状态转移：

```text
dp[i][state] = 从 dp[i - 1][other_state] 或更早状态转移
```

#### 第五步：初始化边界

- 初始化一定要从状态定义反推，不能硬背。
- 常见边界包括：
  - `dp[0] = 0`
  - `dp[1] = ...`
  - `dp[i]` 初值设为只选自己或单独成段
  - 不合法状态设为极小值或极大值

#### 第六步：确定答案位置

- 有时答案是：

```text
dp[n]
```

- 有时答案是：

```text
dp[n - 1]
```

- 还有些题答案是：

```text
max(dp)
```

- 所以一定要根据状态含义判断，而不是默认取最后一个状态。

### 2. 线性 dp 的通用题解口令

- 做题时可以先在脑子里过这几句话：
  - `定位置`
  - `定状态`
  - `看选择`
  - `找来源`
  - `设边界`
  - `定答案`

- 也可以用一句更口语化的话记：

```text
先想 dp[i] 表示什么，再想第 i 个位置怎么由前面位置推出
```

### 3. 基础模板一：前缀型线性 dp

```python
n = len(nums)
dp = [0] * (n + 1)

# 初始化
dp[0] = 0

for i in range(1, n + 1):
    # 根据题意转移
    # dp[i] = ...
    pass

print(dp[n])
```

- 适用场景：
  - 前 `i` 个元素的最优值
  - 前 `i` 个元素的方案数
  - 前缀划分类问题

### 4. 基础模板二：以 i 结尾型线性 dp

```python
n = len(nums)
dp = [0] * n

for i in range(n):
    dp[i] = 初始值
    for j in range(i):
        # 如果 j 能转移到 i
        # dp[i] = max(dp[i], dp[j] + ...)
        pass

print(max(dp))
```

- 适用场景：
  - 最长上升子序列
  - 最大整除子集
  - 以当前位置结尾的最优值问题

### 5. 基础模板三：双状态或多状态线性 dp

```python
n = len(nums)
dp = [[0] * 状态数 for _ in range(n)]

# 初始化第 0 个位置
dp[0][0] = ...
dp[0][1] = ...

for i in range(1, n):
    dp[i][0] = ...
    dp[i][1] = ...
    # 根据状态转移

print(答案)
```

- 适用场景：
  - 股票问题
  - 删除一次、持有/不持有、选/不选等状态机问题

### 6. 基础模板四：滚动变量优化

```python
prev2 = ...
prev1 = ...

for i in range(2, n + 1):
    cur = ...
    prev2, prev1 = prev1, cur

print(prev1)
```

- 当 `dp[i]` 只依赖前面少量状态时，通常可以把空间从 `O(n)` 优化到 `O(1)`。

### 7. 做题时的固定自问清单

- `dp[i]` 表示前缀最优，还是以 `i` 结尾
- 第 `i` 个位置有哪些决策
- 当前状态从哪些旧状态转移而来
- 初始化应该怎么设
- 答案是 `dp[n]`、`dp[n - 1]` 还是 `max(dp)`
- 能不能做滚动优化

### 8. 线性 dp 的常见错误

- 把“前 `i` 个”的定义和“以 `i` 结尾”的定义混在一起。
- 忘记讨论“不选当前元素”。
- 忘记讨论“从当前位置重新开始”。
- 初始化错误，尤其是 `i = 0` 和 `i = 1` 的边界。
- 多状态题没有把非法状态设成极小值或极大值。
- 答案位置取错，本来应该取 `max(dp)` 却直接返回最后一个状态。
- 空间优化时覆盖了后面还要用的旧值。

## 四、线性 DP 题单 + 每类题的代表题和题解套路

### 1. 简单递推类

- `特点`：
  - `dp[i]` 只依赖前面 1 到 2 个状态。
  - 往往围绕“最后一步从哪里来”展开。

- `代表题`：
  - `509. 斐波那契数`
  - `70. 爬楼梯`
  - `746. 使用最小花费爬楼梯`

- `题解套路`：
  - 定义 `dp[i]` 表示到达第 `i` 个位置时的答案。
  - 讨论最后一步是从 `i - 1` 来，还是从 `i - 2` 来。
  - 再把两种来源做求和、取最小值或取最大值。

- `典型转移`：

```text
dp[i] = dp[i - 1] + dp[i - 2]
dp[i] = min(dp[i - 1], dp[i - 2]) + cost[i]
```

- `训练重点`：
  - 入门状态定义
  - 简单边界初始化
  - 滚动数组优化

### 2. 选或不选类

- `特点`：
  - 每个位置通常只有“选”和“不选”两种决策。
  - 当前位置的选择，会影响相邻位置是否还能选。

- `代表题`：
  - `198. 打家劫舍`
  - `213. 打家劫舍 II`
  - `740. 删除并获得点数`

- `题解套路`：
  - 定义 `dp[i]` 表示考虑前 `i` 个位置时的最优值。
  - 对第 `i` 个位置分类讨论：
    - 不选它，则答案来自前一个状态
    - 选它，则答案来自更早的可兼容状态

- `典型转移`：

```text
dp[i] = max(dp[i - 1], dp[i - 2] + nums[i])
```

- `训练重点`：
  - “选 / 不选”建模
  - 环形变形题处理
  - 把原题转换成标准相邻约束模型

### 3. 以 i 结尾类

- `特点`：
  - 状态强调“必须以当前位置结尾”。
  - 常见于连续段最优值问题。

- `代表题`：
  - `53. 最大子数组和`
  - `152. 乘积最大子数组`
  - `32. 最长有效括号`

- `题解套路`：
  - 定义 `dp[i]` 表示以第 `i` 个位置结尾的最优值。
  - 思考当前位置是：
    - 接在前面的结果后面
    - 还是从当前位置重新开始

- `典型转移`：

```text
dp[i] = max(nums[i], dp[i - 1] + nums[i])
```

- `训练重点`：
  - 理解“以 `i` 结尾”这一类状态
  - 分清局部最优和全局最优
  - 学会从 `max(dp)` 取答案

### 4. 枚举前驱类

- `特点`：
  - `dp[i]` 往往要从所有 `j < i` 的位置中选择最优前驱。
  - 常见于子序列类问题。

- `代表题`：
  - `300. 最长上升子序列`
  - `376. 摆动序列`
  - `646. 最长数对链`
  - `368. 最大整除子集`

- `题解套路`：
  - 定义 `dp[i]` 表示以 `i` 结尾的最优值。
  - 枚举所有 `j < i`。
  - 如果 `j` 可以接到 `i`，就尝试用 `dp[j]` 更新 `dp[i]`。

- `典型转移`：

```text
dp[i] = max(dp[i], dp[j] + 1), j < i
```

- `训练重点`：
  - 枚举前驱
  - “谁能接到我”这种思维方式
  - 理解 `O(n^2)` 线性 dp 的经典结构

### 5. 多状态状态机类

- `特点`：
  - 每个位置不止一个状态。
  - 常见于有行为限制、次数限制、持有状态限制的问题。

- `代表题`：
  - `121. 买卖股票的最佳时机`
  - `122. 买卖股票的最佳时机 II`
  - `123. 买卖股票的最佳时机 III`
  - `309. 最佳买卖股票时机含冷冻期`
  - `714. 买卖股票的最佳时机含手续费`
  - `1186. 删除一次得到子数组最大和`

- `题解套路`：
  - 先列清楚每个位置可能处于哪些状态。
  - 再定义 `dp[i][state]`。
  - 最后根据“今天什么都不做 / 买入 / 卖出 / 删除 / 保持”来写状态转移。

- `股票通用定义`：

```text
dp[i][0] = 第 i 天结束时不持股的最大收益
dp[i][1] = 第 i 天结束时持股的最大收益
```

- `股票通用转移`：

```text
dp[i][0] = max(dp[i - 1][0], dp[i - 1][1] + prices[i])
dp[i][1] = max(dp[i - 1][1], 买入来源)
```

- `训练重点`：
  - 画状态图
  - 合法状态转移
  - 非法状态初始化

### 6. 方案数类

- `特点`：
  - 题目问的不是最大值或最小值，而是“有多少种方法”。
  - 状态转移通常是累加。

- `代表题`：
  - `70. 爬楼梯`
  - `91. 解码方法`
  - `377. 组合总和 IV`

- `题解套路`：
  - 定义 `dp[i]` 表示处理到第 `i` 个位置时的方案数。
  - 枚举最后一步或最后一段的来源。
  - 如果当前选择合法，就把来源状态的方案数加到当前状态上。

- `典型转移`：

```text
dp[i] += dp[i - 1]
dp[i] += dp[i - 2]
```

- `训练重点`：
  - 方案数状态定义
  - 合法性判断
  - 前导零、非法组合等边界处理

### 7. 前缀划分类

- `特点`：
  - 前缀答案由“最后一段”决定。
  - 当前状态往往需要枚举断点。

- `代表题`：
  - `139. 单词拆分`
  - `279. 完全平方数`
  - `343. 整数拆分`
  - `1043. 分隔数组以得到最大和`

- `题解套路`：
  - 定义 `dp[i]` 表示前 `i` 个元素的答案。
  - 枚举最后一段从哪里开始，也就是枚举断点 `j`。
  - 把前面一段的答案和最后一段的贡献合并起来。

- `典型转移`：

```text
dp[i] = min/max(dp[j] + cost(j, i))
dp[i] = dp[j] and valid(j, i)
```

- `训练重点`：
  - “最后一段”视角
  - 枚举断点
  - 区间合法性预处理

### 8. 推荐题单

#### 1. 入门必刷

- `509. 斐波那契数`
- `70. 爬楼梯`
- `746. 使用最小花费爬楼梯`
- `198. 打家劫舍`
- `53. 最大子数组和`

- 这一组主要训练：
  - 状态定义
  - 基本转移
  - 边界初始化
  - 空间优化

#### 2. 基础进阶

- `213. 打家劫舍 II`
- `300. 最长上升子序列`
- `91. 解码方法`
- `139. 单词拆分`
- `32. 最长有效括号`

- 这一组主要训练：
  - 结尾型状态
  - 枚举前驱
  - 前缀划分

## 五、总结

- `线性 dp` 的本质，是按顺序处理每个位置，并用前面状态的答案递推出当前位置的答案。
- 它主要解决的是：`和顺序有关的最优值、最少代价、最长长度、方案数和有限状态决策问题`。
- 判断一道题是否适合线性 dp，可以重点看四点：
  - 是否存在明显的线性顺序
  - 是否能定义“处理到第 i 个位置”的状态
  - 当前状态是否依赖前面状态
  - 是否存在重复子问题
- 做线性 dp 时最关键的，不是背公式，而是：

```text
先想清楚状态含义，再围绕当前位置的选择写转移
```

- 你可以把线性 dp 记成一句话：

```text
按顺序推进，记录历史最优，用前面的答案推出当前答案
```
