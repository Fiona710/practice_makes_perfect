# LeetCode 动态规划-区间总结

## 一、区间动态规划的概念

### 1. 什么是区间动态规划

- `区间动态规划`，通常简称 `区间 dp`，指的是把一个连续区间当成状态来做动态规划。
- 这类题最常见的状态定义是：

```text
dp[i][j]
```

- 它表示区间 `[i, j]` 这一段的某种答案。
- 这里的“答案”可能是：
  - 这个区间是否满足某种性质
  - 这个区间上的最优值
  - 这个区间的最少代价
  - 这个区间的方案数

### 2. 为什么它叫“区间”动态规划

- 因为这类题研究的对象，不是“前 `i` 个元素”，而是“从 `i` 到 `j` 的这一段连续部分”。
- 对字符串来说，通常对应一个`子串`。
- 对数组来说，通常对应一个`子数组`或一个待处理的连续区间。
- 一个区间天然由两个端点决定，所以状态通常是二维的。

### 3. 区间 dp 的核心思想

- 先把原问题拆成很多更小的区间问题。
- 再利用小区间的答案，逐步推出大区间的答案。
- 本质上是在做一句话：

```text
大区间的答案，依赖更小区间的答案
```

- 因此，区间 dp 的关键不只是会写状态，还要会判断：
  - 当前区间依赖哪些更小区间
  - 这些更小区间应该先怎么算出来

### 4. 区间 dp 和普通一维 dp 的区别

- 一维 `dp` 更常见于前缀型问题，比如：
  - 走到第 `i` 个位置的最优解
  - 前 `i` 个元素的最优解
- 区间 `dp` 更适合处理“某一整段内部如何演化”的问题。
- 所以区间 dp 的核心视角不是“走到哪里”，而是“这一段区间本身怎么样”。

### 5. 最常见的状态定义方式

- 判断型：

```text
dp[i][j] = s[i...j] 是否满足条件
```

- 最优值型：

```text
dp[i][j] = 区间 [i, j] 上的最大值 / 最小值
```

- 计数型：

```text
dp[i][j] = 区间 [i, j] 上满足条件的方案数
```

### 6. 区间 dp 的思考顺序

- 第一步：定义 `dp[i][j]` 表示什么
- 第二步：想当前区间 `[i, j]` 能否由更小区间推出
- 第三步：确定最小区间的边界条件
- 第四步：根据依赖关系确定遍历顺序
- 第五步：在填表过程中更新答案

可以把它总结成一句口令：

```text
先定区间状态，再想小区间如何推出大区间
```

## 二、区间动态规划解决的问题

### 1. 这类题通常在解决什么

- 区间 dp 主要解决的是：`关于一个连续区间的性质、代价、最优值或组合方式`。
- 题目虽然表面不同，但本质上常见有下面几类：
  - 判断一个区间是否满足某种条件
  - 求一个区间上的最优结果
  - 求处理一个区间的最少代价
  - 统计一个区间的可行方案数

### 2. 判断区间性质的问题

- 典型代表是：
  - `5. 最长回文子串`
  - `647. 回文子串`
- 这类题常见做法是定义：

```text
dp[i][j] = 区间 [i, j] 是否满足某种性质
```

- 例如在回文问题中：

```text
dp[i][j] = s[i...j] 是否是回文串
```

- 当前区间往往依赖内部更小区间：

```text
dp[i][j] <- dp[i + 1][j - 1]
```

### 3. 求区间最优值的问题

- 这类题会问：
  - 最长
  - 最少
  - 最大
  - 最小
- 典型代表是：
  - `516. 最长回文子序列`
  - `312. 戳气球`
  - `1039. 多边形三角剖分的最低得分`
- 这类题的状态通常定义成：

```text
dp[i][j] = 处理区间 [i, j] 的最优答案
```

### 4. 求区间最少代价的问题

- 这类题本质是：把一个大区间通过某些操作处理完成，求代价最小。
- 常见特征是：
  - 可以在区间内切分
  - 每种切分方式代价不同
  - 需要从所有切分方式里选最优
- 常见转移形式：

```text
dp[i][j] = min(dp[i][k] + dp[k + 1][j] + cost)
```

- 典型例子：
  - `1547. 切棍子的最小成本`
  - `1000. 合并石头的最低成本`

### 5. 统计区间方案数的问题

- 有些题不是问最优值，而是问：
  - 有多少种合法处理方式
  - 某个区间能组成多少种结果
- 这类题也可以用区间 dp，只是 `dp[i][j]` 表示的是方案数。

## 三、区间动态规划的场景特征

### 1. 什么时候要优先想到区间 dp

- 当题目研究对象是`连续的一段`时，要优先往区间 dp 上想。
- 特别是题目里出现这些词时，要提高敏感度：
  - 子串
  - 子数组
  - 区间
  - 合并
  - 切分
  - 回文

### 2. 典型识别特征

- `特征一：状态天然由两个端点决定`
  - 如果一个子问题必须由左端点 `i` 和右端点 `j` 才能描述清楚，往往就是区间 dp。

- `特征二：大问题可以缩成更小的同类区间问题`
  - 例如去掉两端后，变成内部区间
  - 或者在中间切一刀，变成左右两个子区间

- `特征三：题目强调连续性`
  - 子串和子数组要求连续，这和前缀 dp、子序列 dp 很不一样。

- `特征四：答案和区间内部结构有关`
  - 不是只和某个位置有关，而是和整段内部怎么组合、怎么分裂、怎么匹配有关。

### 3. 区间 dp 最常见的两类转移

#### 1. 两端收缩型

- 当前区间通过“看两端 + 依赖内部区间”来转移。
- 常见形式：

```text
dp[i][j] <- dp[i + 1][j - 1]
```

- 适合的问题：
  - 回文判断
  - 回文相关最优值
  - 两端字符匹配问题

- `最长回文子串` 就是这一类的典型代表：

```text
dp[i][j] = (s[i] == s[j]) && (j - i <= 2 || dp[i + 1][j - 1])
```

#### 2. 中间切分型

- 当前区间通过枚举分割点 `k`，拆成左右两个区间。
- 常见形式：

```text
dp[i][j] = min/max(dp[i][k] + dp[k + 1][j])
```

- 有时还会附带当前这次切分的代价：

```text
dp[i][j] = min/max(dp[i][k] + dp[k + 1][j] + cost)
```

- 适合的问题：
  - 区间合并
  - 区间分割
  - 括号、表达式、石子合并一类问题

### 4. 遍历顺序为什么重要

- 区间 dp 最大的实现关键，是`必须先算小区间，再算大区间`。
- 因为大区间依赖的小区间如果还没算出来，当前状态就无法转移。
- 最稳妥的写法通常是按区间长度从小到大遍历：

```python
for length in range(1, n + 1):
    for i in range(0, n - length + 1):
        j = i + length - 1
        # 计算 dp[i][j]
```

- 这个顺序的本质是：`先处理短区间，再处理长区间`。

### 5. 做题时的固定自问清单

- `dp[i][j]` 到底表示什么
- 当前区间的最后一步是什么
- 是依赖内部区间，还是依赖左右切分
- 最小区间是什么，边界怎么设
- 遍历顺序怎样才能保证依赖先被算出

## 四、区间动态规划的同类型题目

### 1. 回文区间类

- `5. 最长回文子串`
- `647. 回文子串`
- `516. 最长回文子序列`

这类题的特点是：

- 常常围绕两端字符是否相等展开
- 状态通常和 `dp[i + 1][j - 1]` 有关
- 适合训练“区间收缩”的思路

### 2. 切分与合并类

- `312. 戳气球`
- `1547. 切棍子的最小成本`
- `1000. 合并石头的最低成本`
- `1039. 多边形三角剖分的最低得分`

这类题的特点是：

- 通常需要枚举中间切分点 `k`
- 当前答案往往来自多个子区间结果的组合
- 适合训练“枚举分割点”的思路

### 3. 表达式与结构划分类

- `241. 为运算表达式设计优先级`
- 一些括号匹配、表达式拆分、结构组合类题目

这类题的特点是：

- 需要把整个区间拆成左右两部分
- 不同拆法对应不同结果
- 区间 dp 往往和递归划分思路结合得很紧

### 4. 建议的练习顺序

建议按下面顺序练习区间 dp：

1. `5. 最长回文子串`
2. `647. 回文子串`
3. `516. 最长回文子序列`
4. `312. 戳气球`
5. `1547. 切棍子的最小成本`
6. `1039. 多边形三角剖分的最低得分`

这样安排的原因是：

- 先从回文类题目理解“区间状态”和“长度遍历”
- 再过渡到更典型的切分型区间 dp
- 最后形成对区间收缩和区间分裂两种模型的完整认识

## 五、总结

- `区间动态规划` 的本质，是把一个连续区间 `[i, j]` 作为状态，利用更小区间的答案推出更大区间的答案。
- 它主要解决的是：`连续区间上的性质判断、最优值计算、最少代价、方案统计`。
- 判断是否适合用区间 dp，可以重点看四点：
  - 是否研究连续区间
  - 是否能用两个端点描述状态
  - 是否能由更小区间推出大区间
  - 是否存在两端收缩或中间切分的结构
- 区间 dp 最关键的实现点，不是背公式，而是：

```text
先想清楚状态依赖，再决定遍历顺序
```

- 你可以把区间 dp 记成一句话：

```text
研究一整段，拆成更小段，先算小区间，再推大区间
```
