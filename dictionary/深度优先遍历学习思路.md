# 深度优先遍历学习思路

## 对话背景

### 用户问题一

> Java程序员跳槽找工作，现在在刷LeetCode上的题目，对深度优先遍历算法不了解，请给出学习深度优先遍历算法的辅导建议。

### 回复内容一

## 学习重点

- 把深度优先遍历 `DFS` 理解成一种“沿着一条路先走到底，再回头换路”的搜索方式。
- 对 Java 面试和 `LeetCode` 来说，`DFS` 最常见的载体有 3 类：`二叉树`、`图`、`回溯题`。
- 你现在不需要一开始就学很复杂的图论，建议先从 `二叉树 DFS` 入手，再过渡到 `网格/图`，最后学 `回溯`。

## 先建立直觉

- `BFS` 像一层一层找人，`DFS` 像走迷宫时先认准一条路一直走。
- `DFS` 的核心动作只有两个：`进入当前节点`、`递归访问下一个节点`。
- 真正难点不是代码，而是想清楚“当前这一层函数负责什么”。

## 你要先学会的 4 个问题

- 当前函数的参数是什么：比如 `dfs(TreeNode node)` 或 `dfs(int i, int j)`。
- 递归终止条件是什么：比如节点为空、越界、已经访问过。
- 当前层要做什么：统计、判断、记录路径、更新答案。
- 下一层怎么走：访问左子树右子树，或上下左右。

## 推荐学习顺序

- 第 1 步：学 `二叉树前序/中序/后序遍历`，这是 DFS 最基础的形态。
- 第 2 步：学 `二叉树路径问题`，比如求最大深度、路径和。
- 第 3 步：学 `网格 DFS`，比如岛屿数量、矩阵中的连通块。
- 第 4 步：学 `图 DFS`，重点理解 `visited` 防止重复访问。
- 第 5 步：学 `回溯`，本质上也是 DFS，只是多了“做选择 + 撤销选择”。

## Java 程序员最适合掌握的 DFS 模板

- 二叉树模板：

```java
void dfs(TreeNode node) {
    if (node == null) {
        return;
    }

    // 处理当前节点

    dfs(node.left);
    dfs(node.right);
}
```

- 网格模板：

```java
void dfs(char[][] grid, int i, int j) {
    if (i < 0 || i >= grid.length || j < 0 || j >= grid[0].length) {
        return;
    }
    if (grid[i][j] != '1') {
        return;
    }

    grid[i][j] = '0'; // 标记已访问

    dfs(grid, i + 1, j);
    dfs(grid, i - 1, j);
    dfs(grid, i, j + 1);
    dfs(grid, i, j - 1);
}
```

- 回溯模板：

```java
void dfs(List<Integer> path, int[] nums, boolean[] used) {
    if (path.size() == nums.length) {
        // 记录答案
        return;
    }

    for (int i = 0; i < nums.length; i++) {
        if (used[i]) {
            continue;
        }

        used[i] = true;
        path.add(nums[i]);

        dfs(path, nums, used);

        path.remove(path.size() - 1);
        used[i] = false;
    }
}
```

## 刷题时的思考套路

- 先问自己：题目是在“遍历所有可能状态”吗？如果是，优先想 `DFS`。
- 再问：每次递归函数代表什么状态？
- 再问：什么时候停止？
- 最后问：当前层要不要恢复现场，也就是是否需要“回溯”。

## 你最容易踩的坑

- 忘记写终止条件，导致死递归。
- 图和网格题忘记 `visited` 或标记访问。
- 回溯题忘记撤销状态，比如 `remove`、`used[i] = false`。
- 把“当前层做什么”和“子问题做什么”混在一起，代码会很乱。
- 递归返回值设计不清晰，比如有的题需要 `boolean`，有的题需要 `int`，有的题直接改全局变量。

## 适合 Java 程序员的学习方法

- 不要死背题解，先自己用中文写出“递归函数定义”。
- 每道题都先写一句话：`dfs(x) 表示什么`。
- 手动画递归调用栈，尤其是树和回溯题，效果非常好。
- 同一题尽量自己写两遍：第一遍照模板写，第二遍脱离模板写。
- 把题目归类到固定模式：`树遍历`、`连通块`、`枚举所有方案`。

## 一个很关键的认知

- `回溯 = 带撤销操作的 DFS`
- 所以你一旦学会普通 DFS，再学排列、组合、子集会顺很多。

## 建议你这样安排 7 天入门

- 第 1 天：理解递归和二叉树前中后序。
- 第 2 天：做 2 到 3 道树的 DFS 基础题。
- 第 3 天：做网格 DFS，重点练越界判断和访问标记。
- 第 4 天：复习图的 `visited` 思想。
- 第 5 天：学回溯模板。
- 第 6 天：做排列、组合、子集。
- 第 7 天：总结 DFS 题型模板，整理自己的错题笔记。

## 给你的实用建议

- 你是 Java 程序员，优势在于代码规范和抽象能力，建议把 DFS 总结成自己的模板库。
- 跳槽面试中，DFS 不只是考算法，也考你能不能把递归思路讲清楚。
- 面试时可以先说：
  - “我把这题抽象成一棵搜索树。”
  - “每层递归表示当前处理到哪个状态。”
  - “终止条件是……”
  - “然后枚举下一步决策，递归深入。”

## 学习路线图

- 目标分三层：先理解 `DFS` 递归框架，再掌握 `树/网格/图` 的访问方式，最后过渡到 `回溯`。
- 学习顺序建议：`递归基础 -> 二叉树 DFS -> 网格 DFS -> 图 DFS -> 回溯`
- 每道题都先写一句话：`dfs(...)` 的定义是什么。
- 每次做题固定检查 4 件事：`参数`、`终止条件`、`当前层处理`、`下一层递归`。
- 不要一开始追求“秒杀题”，先把同一类题做出稳定模板感。

## 第一阶段：先把 DFS 学明白

- 核心理解：`DFS` 就是“当前节点处理完，再递归处理子问题”。
- 最基础模板：

```java
void dfs(Node node) {
    if (node == null) {
        return;
    }

    // 处理当前节点

    for (Node next : node.children) {
        dfs(next);
    }
}
```

- 你要真正想清楚的是：
  - 这个函数的职责是什么
  - 什么时候返回
  - 返回值要不要带信息
  - 是否需要记录访问状态 `visited`
  - 是否需要“撤销选择”，也就是回溯

## 每题讲解重点

- `104. 二叉树的最大深度`
  - 训练目标：理解“函数返回值型 DFS”
  - 关键定义：`dfs(node)` 表示以 `node` 为根的树的最大深度
  - 重点：学会写 `return Math.max(left, right) + 1`
  - 易错点：空节点返回 `0`

- `100. 相同的树`
  - 训练目标：理解“同时递归两棵树”
  - 关键定义：`dfs(p, q)` 表示两棵子树是否相同
  - 重点：先处理空节点，再比较值，再递归左右子树
  - 易错点：空节点判断顺序写乱

- `112. 路径总和`
  - 训练目标：理解“路径状态向下传递”
  - 关键定义：`dfs(node, target)` 表示从当前节点往下是否存在满足条件的路径
  - 重点：参数里带状态，通常比全局变量更清晰
  - 易错点：只在叶子节点判断是否满足

- `543. 二叉树的直径`
  - 训练目标：理解“递归返回一个值，同时更新全局答案”
  - 关键定义：`dfs(node)` 返回以 `node` 为根的最大深度
  - 重点：当前节点的直径候选值是 `leftDepth + rightDepth`
  - 易错点：把“深度”和“直径”混成一个概念

- `200. 岛屿数量`
  - 训练目标：理解“网格 DFS + 访问标记”
  - 关键定义：`dfs(i, j)` 表示淹没从 `(i, j)` 出发的整块陆地
  - 重点：遍历网格时，遇到陆地就 `count++`，然后 DFS 清掉整块
  - 易错点：越界判断、重复访问、方向数组写错

- `695. 岛屿的最大面积`
  - 训练目标：理解“DFS 返回连通块大小”
  - 关键定义：`dfs(i, j)` 返回当前岛屿面积
  - 重点：当前格子记作 `1`，加上四个方向的递归结果
  - 易错点：访问标记不及时，导致重复累计

- `733. 图像渲染`
  - 训练目标：进一步巩固网格 DFS
  - 关键定义：从起点出发，把所有“颜色相同且连通”的格子改色
  - 重点：先判断新旧颜色是否相同，相同就直接返回
  - 易错点：如果不先判断颜色相同，会死循环或重复递归

- `46. 全排列`
  - 训练目标：正式进入回溯
  - 关键定义：`dfs(path, used)` 表示当前已经选了 `path`，继续枚举后续位置
  - 重点：掌握标准回溯结构：`选择 -> 递归 -> 撤销`
  - 易错点：忘记 `used[i] = false`，忘记删除路径最后一个元素

- `77. 组合`
  - 训练目标：理解“从某个起点开始枚举”
  - 关键定义：`dfs(start, path)` 表示从数字 `start` 开始选，构造组合
  - 重点：用 `start` 避免重复选择
  - 易错点：循环起点没处理好，导致重复组合

- `39. 组合总和`
  - 训练目标：理解“可重复选元素”的回溯
  - 关键定义：`dfs(start, remain, path)` 表示从 `start` 开始选，剩余和为 `remain`
  - 重点：同一元素可重复使用，所以递归时仍传 `i` 而不是 `i + 1`
  - 易错点：剪枝条件不清楚，导致效率差或漏解

## 你做题时的固定模板

- 第一步：先写出函数定义
  - 例子：`dfs(node)` 到底表示什么？
- 第二步：写终止条件
  - 空节点、越界、访问过、剩余值不合法
- 第三步：写当前层逻辑
  - 更新答案、加入路径、标记访问
- 第四步：递归下一层
- 第五步：如果是回溯题，撤销现场

## Java 写 DFS 的常用模板

- 树的返回值 DFS：

```java
public int maxDepth(TreeNode root) {
    if (root == null) {
        return 0;
    }
    return Math.max(maxDepth(root.left), maxDepth(root.right)) + 1;
}
```

- 网格 DFS：

```java
void dfs(char[][] grid, int i, int j) {
    if (i < 0 || i >= grid.length || j < 0 || j >= grid[0].length) {
        return;
    }
    if (grid[i][j] != '1') {
        return;
    }

    grid[i][j] = '0';

    dfs(grid, i + 1, j);
    dfs(grid, i - 1, j);
    dfs(grid, i, j + 1);
    dfs(grid, i, j - 1);
}
```

- 回溯 DFS：

```java
void dfs(int[] nums, boolean[] used, List<Integer> path, List<List<Integer>> ans) {
    if (path.size() == nums.length) {
        ans.add(new ArrayList<>(path));
        return;
    }

    for (int i = 0; i < nums.length; i++) {
        if (used[i]) {
            continue;
        }

        used[i] = true;
        path.add(nums[i]);

        dfs(nums, used, path, ans);

        path.remove(path.size() - 1);
        used[i] = false;
    }
}
```

## 高频误区

- 把递归函数定义说不清，导致代码越写越乱
- 树题里返回值和全局变量混用，没有分工
- 网格题漏掉 `visited` 或原地标记
- 回溯题忘记撤销状态
- 组合题和排列题没有分清
  - 排列：看顺序，常用 `used[]`
  - 组合：不看顺序，常用 `start`
