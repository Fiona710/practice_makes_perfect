# LeetCode 动态规划-背包总结

## 一、背包动态规划的概念

### 1. 什么是背包动态规划

- `背包动态规划`，通常简称 `背包 dp`，是一类围绕“在有限容量或有限资源下做选择”的动态规划问题。
- 这类题一般会有两部分信息：
  - 若干个可供选择的元素
  - 一个总限制条件
- 这些元素可以是：
  - 物品
  - 数字
  - 硬币
  - 字符串
  - 任务
- 这个限制条件可以是：
  - 容量
  - 重量
  - 金额
  - 时间
  - 个数
  - 目标和

### 2. 为什么它叫“背包”动态规划

- 因为最经典的模型是：
  - 有一个容量为 `m` 的背包
  - 有若干件物品，每件物品有 `体积` 和 `价值`
  - 问怎样选择物品，才能让总价值最大
- 但在实际题目中，“背包”通常只是一个抽象说法。
- 题目不一定真的出现背包，而是表现为：
  - 从数组里选一些数，使和不超过某个值
  - 用若干硬币凑出某个金额
  - 选若干字符串，使资源消耗不超限
  - 在限制条件下求最大值、最小值、可行性或方案数

### 3. 背包 dp 的核心思想

- 把原问题拆成很多“考虑前若干个元素，在某个容量限制下的子问题”。
- 用更小规模的状态，逐步推出更大规模的状态。
- 本质上是在回答一句话：

```text
在容量或资源限制下，当前元素选还是不选，选几次，能得到什么答案
```

- 所以背包 dp 的关键不只是会写转移式，更重要的是先想清楚：
  - 状态表示什么
  - 当前元素能选几次
  - 是否要求恰好装满
  - 题目求的是最值、可行性还是方案数
  - 容量这一维该怎么遍历

### 4. 背包 dp 和其他 dp 的区别

- `线性 dp` 更关注位置顺序，比如“处理到第 `i` 个位置”。
- `区间 dp` 更关注连续区间 `[i, j]`。
- `双序列 dp` 更关注两个序列前缀之间的关系。
- `背包 dp` 更关注：
  - 选哪些元素
  - 消耗多少资源
  - 在限制下能得到什么结果

### 5. 背包 dp 最常见的状态定义

- 二维写法：

```text
dp[i][j] = 前 i 个物品里，容量为 j 时的答案
```

- 一维优化写法：

```text
dp[j] = 容量为 j 时的答案
```

- 这里的“答案”可能是：
  - 最大价值
  - 最小代价
  - 是否可行
  - 方案数

### 6. 背包 dp 的思考顺序

- 第一步：确定“物品”是什么
- 第二步：确定“容量”或“限制”是什么
- 第三步：判断每个物品能选几次
- 第四步：定义 `dp` 状态
- 第五步：判断题目求的是最值、可行性还是方案数
- 第六步：根据题意设置初始化
- 第七步：根据物品使用次数决定容量遍历顺序

- 可以把它总结成一句口令：

```text
先定物品和容量，再定状态，再判断当前物品能不能重复使用
```

## 二、背包动态规划的典型特征

### 1. 什么时候要优先想到背包 dp

- 当题目本质是在“若干元素中做选择”，并且存在某种总资源限制时，要优先想到背包 dp。
- 特别是题目里出现下面这些词时，要提高敏感度：
  - 选择若干个数
  - 每个数只能用一次
  - 每个数可以重复使用
  - 容量
  - 金额
  - 目标和
  - 恰好凑成
  - 最多
  - 最少
  - 方案数

### 2. 背包 dp 的典型识别特征

- `特征一：有资源限制`
  - 比如容量、金额、时间、个数、和、0 的数量、1 的数量等。

- `特征二：核心决策是选择元素`
  - 当前元素选还是不选，选几次，决定了资源消耗和答案变化。

- `特征三：子问题可以表示为“考虑前几个元素 + 当前容量”`
  - 这是最典型的背包建模方式。

- `特征四：答案可能是最值、可行性或方案数`
  - 一道背包题不一定求最大价值，也可能问能不能做到，或者有多少种方法。

- `特征五：题面不一定出现背包`
  - 很多题其实是“子集和”“零钱兑换”“目标和”“分组选择”这类伪装形式。

### 3. 背包 dp 中“容量”的常见伪装

- 体积或重量
- 金额
- 目标和
- 时间
- 可选次数
- 字符串中 `0` 和 `1` 的数量
- 差值、剩余量、预算

### 4. 背包 dp 最核心的两种决策

- `不选当前物品`
  - 当前状态直接继承旧答案

- `选当前物品`
  - 当前状态由更小容量的状态转移而来

- 对很多题来说，真正的本质就是一句话：

```text
当前元素不参与，还是参与到当前容量的构造中
```

## 三、背包动态规划的常见分类

### 1. 0/1 背包

- 每个物品最多只能选一次。
- 最经典的转移是：

```text
dp[i][j] = max(dp[i - 1][j], dp[i - 1][j - w[i]] + v[i])
```

- 一维优化后常写成：

```text
dp[j] = max(dp[j], dp[j - w] + v)
```

- 容量 `j` 必须倒序遍历。
- 原因是：防止同一轮里把同一个物品重复使用多次。

### 2. 完全背包

- 每个物品可以选无限次。
- 一维转移通常写成：

```text
dp[j] = max(dp[j], dp[j - w] + v)
```

- 容量 `j` 必须正序遍历。
- 原因是：要允许当前物品在本轮继续被重复使用。

### 3. 多重背包

- 每个物品最多可以选 `k` 次。
- 常见做法有两种：
  - 直接枚举选几次
  - 用二进制拆分，把一个物品拆成多个 `0/1` 物品

### 4. 分组背包

- 物品被分成若干组，每组最多选一个。
- 常见于：
  - 套餐选择
  - 型号选择
  - 每组选一个方案
- 转移时通常是：
  - 先枚举组
  - 再枚举容量
  - 再枚举本组里选哪个物品

### 5. 混合背包

- 题目中同时存在：
  - 只能选一次的物品
  - 可以选无限次的物品
  - 有次数上限的物品
- 做法是根据每个物品的类型，分别套对应的遍历方式。

### 6. 多维费用背包

- 不止一个限制条件。
- 例如：
  - 同时限制体积和重量
  - 同时限制 `0` 的数量和 `1` 的数量
- 常见状态形式：

```text
dp[i][j]
dp[x][y]
dp[i][j][k]
```

### 7. 树形背包和依赖背包

- 有些物品之间存在依赖关系。
- 比如：
  - 选附件前必须先选主件
  - 选子节点前必须先选父节点
- 这时普通背包会和树形 dp、分组讨论结合。

## 四、背包 dp 的题目目标分类

### 1. 最值型背包

- 目标是求：
  - 最大价值
  - 最小代价
  - 最多件数
  - 最少硬币数
- 这类题的 `dp` 通常存储“最优值”。

### 2. 可行性背包

- 目标是判断：
  - 能否凑出某个目标
  - 能否恰好装满
  - 能否分成某种结构
- 这类题的 `dp` 往往是布尔类型：

```text
dp[j] = 是否可行
```

### 3. 计数型背包

- 目标是求：
  - 有多少种方法
  - 有多少种选法
  - 有多少种凑法
- 常见转移是：

```text
dp[j] += dp[j - w]
```

### 4. 构造型背包

- 目标不仅是求值，还要恢复具体选择方案。
- 一般需要：
  - 额外记录路径
  - 或从二维 `dp` 表中反推选择过程

## 五、背包 dp 的解题模板思路

### 1. 通用思考流程

#### 第一步：抽象出物品和容量

- 先问自己两个问题：
  - 什么是“物品”
  - 什么是“容量”
- 例如：
  - 在子集和问题里，数组中的每个数是物品，目标和是容量
  - 在零钱兑换问题里，每种硬币是物品，金额是容量
  - 在“一和零”问题里，每个字符串是物品，`0` 和 `1` 的个数是双容量

#### 第二步：判断每个物品能选几次

- 只能选一次：`0/1 背包`
- 可以无限次选：`完全背包`
- 最多选若干次：`多重背包`
- 每组选一个：`分组背包`

#### 第三步：明确题目在求什么

- 求最大值或最小值
- 求是否可行
- 求方案数

#### 第四步：判断是否要求恰好

- “不超过容量” 和 “恰好装满” 的初始化不同。
- 很多题转移式看起来一样，但初始化不一样，答案就会完全不同。

#### 第五步：定义状态

- 二维经典定义：

```text
dp[i][j] = 前 i 个物品，容量为 j 时的答案
```

- 一维优化定义：

```text
dp[j] = 容量为 j 时的答案
```

#### 第六步：写出转移

- 通常围绕两个决策：
  - 不选当前物品
  - 选当前物品

#### 第七步：确定遍历顺序

- `0/1 背包`：容量倒序
- `完全背包`：容量正序
- `计数题`：还要额外判断是组合数还是排列数

### 2. 背包题的固定自问清单

- 当前元素是什么
- 容量限制是什么
- 每个元素能用几次
- `dp[j]` 表示什么
- 是求最大值、最小值、可行性还是方案数
- 是否要求恰好装满
- 容量是正序还是倒序遍历
- 答案取 `dp[target]` 还是取整个数组中的最优值

### 3. 二维基础模板

```python
n = len(items)
dp = [[0] * (bag + 1) for _ in range(n + 1)]

for i in range(1, n + 1):
    w, v = items[i - 1]
    for j in range(bag + 1):
        dp[i][j] = dp[i - 1][j]
        if j >= w:
            dp[i][j] = max(dp[i][j], dp[i - 1][j - w] + v)

print(dp[n][bag])
```

- 适用场景：
  - 初学背包时理解状态定义
  - 需要恢复路径
  - 需要明确区分“上一层”和“当前层”

### 4. 0/1 背包一维模板

```python
dp = [0] * (bag + 1)

for w, v in items:
    for j in range(bag, w - 1, -1):
        dp[j] = max(dp[j], dp[j - w] + v)

print(dp[bag])
```

- 记忆重点：
  - `先物品，后容量`
  - `容量倒序`

### 5. 完全背包一维模板

```python
dp = [0] * (bag + 1)

for w, v in items:
    for j in range(w, bag + 1):
        dp[j] = max(dp[j], dp[j - w] + v)

print(dp[bag])
```

- 记忆重点：
  - `先物品，后容量`
  - `容量正序`

### 6. 可行性 0/1 背包模板

```python
dp = [False] * (target + 1)
dp[0] = True

for x in nums:
    for j in range(target, x - 1, -1):
        dp[j] = dp[j] or dp[j - x]

print(dp[target])
```

### 7. 方案数完全背包模板

```python
dp = [0] * (target + 1)
dp[0] = 1

for coin in coins:
    for j in range(coin, target + 1):
        dp[j] += dp[j - coin]

print(dp[target])
```

### 8. 最小值完全背包模板

```python
INF = 10 ** 18
dp = [INF] * (target + 1)
dp[0] = 0

for coin in coins:
    for j in range(coin, target + 1):
        dp[j] = min(dp[j], dp[j - coin] + 1)

print(dp[target] if dp[target] != INF else -1)
```

## 六、初始化规律总结

### 1. 最大值型，且不要求恰好装满

- 通常初始化为：

```text
dp[j] = 0
```

- 含义是：什么都不选时，价值为 `0`。

### 2. 最大值型，但要求恰好装满

- 通常初始化为：

```text
dp[0] = 0
其余状态 = -inf
```

- 因为除了容量 `0`，其余容量一开始都不可达。

### 3. 最小值型

- 通常初始化为：

```text
dp[0] = 0
其余状态 = +inf
```

### 4. 可行性型

- 通常初始化为：

```text
dp[0] = True
其余状态 = False
```

### 5. 方案数型

- 通常初始化为：

```text
dp[0] = 1
其余状态 = 0
```

- 这里的 `dp[0] = 1` 表示：
  - 凑出 `0` 的方法有 `1` 种
  - 即“什么都不选”这一种方法

## 七、遍历顺序与计数区别

### 1. 为什么 0/1 背包要倒序

- 因为当前物品只能用一次。
- 如果正序遍历，当前轮更新出来的新状态又会继续参与后面的更新，等价于把同一个物品重复使用多次。

### 2. 为什么完全背包要正序

- 因为当前物品可以重复使用。
- 正序遍历可以让本轮新更新的状态继续被后面的更大容量使用。

### 3. 组合数和排列数的区别

- `组合数`：顺序不重要
  - 例如 `1 + 2` 和 `2 + 1` 算同一种

- `排列数`：顺序重要
  - 例如 `1 + 2` 和 `2 + 1` 算两种

### 4. 组合数的常见循环顺序

```python
for item in items:
    for j in range(item, target + 1):
        dp[j] += dp[j - item]
```

- 含义是：先固定物品种类，再扩展容量，不会重复统计顺序。

### 5. 排列数的常见循环顺序

```python
for j in range(1, target + 1):
    for item in items:
        if j >= item:
            dp[j] += dp[j - item]
```

- 含义是：先考虑容量，再枚举最后一步选哪个元素，因此会统计不同顺序。

## 八、背包 dp 的高频题型总结

### 1. 子集和问题

- 本质通常是 `0/1 背包`。
- 常见形式：
  - 从数组中选若干个数，使和等于某个值
  - 从数组中选若干个数，使和尽量接近某个值
- 代表题：
  - `416. 分割等和子集`
  - `1049. 最后一块石头的重量 II`

- `题解套路`：
  - 把每个数字看成一个物品
  - 把目标和看成背包容量
  - 视题意求可行性、最大可达值或方案数

### 2. 目标和变形

- 常见代表题：
  - `494. 目标和`

- `题解套路`：
  - 把加减号问题转成子集和计数问题
  - 常见推导是：

```text
设加号集合和为 p，减号集合和为 n
p - n = target
p + n = sum
=> p = (sum + target) / 2
```

- 最终转成：
  - 从数组中选若干数，求和为 `p` 的方案数

### 3. 零钱兑换问题

- 本质通常是 `完全背包`。
- 代表题：
  - `322. 零钱兑换`
  - `518. 零钱兑换 II`
  - `279. 完全平方数`

- `题解套路`：
  - 每种硬币或平方数都可以重复使用
  - 根据题意决定求：
    - 最少个数
    - 方案数

### 4. 恰好装满问题

- 题目中常见关键词：
  - 恰好
  - 正好
  - 刚好凑成
- 难点往往不在转移，而在初始化。

- `题解套路`：
  - 如果要求恰好，必须把不可达状态初始化成无穷或负无穷
  - 不能简单全部初始化成 `0`

### 5. 多维费用问题

- 代表题：
  - `474. 一和零`

- `题解套路`：
  - 每个字符串消耗两个资源：`0` 的数量和 `1` 的数量
  - 状态常定义成：

```text
dp[i][j] = 在最多 i 个 0 和 j 个 1 的限制下，最多能选多少个字符串
```

### 6. 分组选择问题

- 常见形式：
  - 每组物品中最多选一个
  - 每组选一个方案使总收益最大

- `题解套路`：
  - 先按组处理
  - 对每一组里的所有选项做一次枚举更新

### 7. 多重背包问题

- 常见形式：
  - 每种物品最多拿若干件
  - 每类资源可使用次数有限

- `题解套路`：
  - 数据小就直接枚举件数
  - 数据大优先考虑二进制拆分优化

### 8. 排列计数问题

- 代表题：
  - `377. 组合总和 IV`

- `题解套路`：
  - 虽然表面上也是“凑目标和”
  - 但题目把不同顺序视作不同方案
  - 因此要用“先容量、后物品”的循环方式统计排列数

## 九、关联题目整理

### 1. 0/1 背包入门题

- `416. 分割等和子集`
- `1049. 最后一块石头的重量 II`
- `494. 目标和`

- 这一组主要训练：
  - 0/1 背包建模
  - 可行性与计数型转化
  - 子集和思维

### 2. 完全背包入门题

- `322. 零钱兑换`
- `518. 零钱兑换 II`
- `279. 完全平方数`

- 这一组主要训练：
  - 完全背包正序更新
  - 最小值型与方案数型的区别
  - 初始化设置

### 3. 多维费用背包题

- `474. 一和零`

- 这一题主要训练：
  - 二维容量设计
  - 二维 0/1 背包倒序更新

### 4. 排列与组合区分题

- `518. 零钱兑换 II`
- `377. 组合总和 IV`

- 这一组主要训练：
  - 组合数与排列数的区别
  - 循环顺序对结果的影响

### 5. 进阶背包题

- 一些主件附件问题
- 一些分组选择问题
- 一些树形依赖背包问题

- 这一组主要训练：
  - 分组思想
  - 依赖关系处理
  - 从普通背包向综合模型迁移

## 十、建议的练习顺序

1. `416. 分割等和子集`
2. `1049. 最后一块石头的重量 II`
3. `322. 零钱兑换`
4. `518. 零钱兑换 II`
5. `279. 完全平方数`
6. `494. 目标和`
7. `474. 一和零`
8. `377. 组合总和 IV`

- 这样安排的原因是：
  - 先掌握最基础的 `0/1 背包`
  - 再掌握 `完全背包`
  - 然后理解计数、最小值和可行性三种目标差异
  - 再过渡到二维费用和排列计数这类变形题

## 十一、背包 dp 的常见错误

- 没分清当前题是 `0/1 背包` 还是 `完全背包`
- `0/1 背包` 把容量写成正序，导致同一物品被重复使用
- `完全背包` 写成倒序，导致无法重复利用当前物品
- 忽略“恰好装满”的要求，初始化写错
- 把“最少个数”和“方案数”用同一套初始化
- 把组合数题写成排列数题，或把排列数题写成组合数题
- `dp[j]` 的含义没有先想清楚，导致转移公式虽然像，但语义不对
- 只会套模板，遇到目标和、分割等和子集这类伪装题就认不出是背包

## 十二、总结

- `背包 dp` 的本质，是在有限资源约束下，从若干元素中做选择。
- 它主要解决的是：
  - 最值问题
  - 可行性问题
  - 方案数问题
- 判断一道题是不是背包 dp，可以重点看四点：
  - 是否是在若干元素中做选择
  - 是否有总资源限制
  - 当前元素是否存在“选 / 不选 / 选几次”的决策
  - 是否能定义“容量状态”

- 做背包题时最关键的，不是死记公式，而是先回答下面几个问题：

```text
物品是什么
容量是什么
每个物品能用几次
题目求什么
是否要求恰好
容量该正序还是倒序
```

- 你可以把背包 dp 记成一句话：

```text
在限制条件下做选择，用小容量状态逐步推出大容量状态
```
