# 回溯算法学习思路

## 问题

Java 程序员刷 LeetCode 题目，对回溯的原理以及实现都不了解，给出学习重点。

## 回复

回溯对刷 LeetCode 很重要，尤其是 `组合`、`排列`、`子集`、`分割`、`N 皇后`、`解数独` 这一类题。你作为 Java 程序员，如果目前对原理和实现都不熟，学习重点建议按下面抓。

### 先理解本质

- 回溯就是一种“试答案 + 不行就撤销”的搜索过程，本质上是 `DFS + 状态撤销`。
- 它适合解决“要求列出所有可能结果”或“从很多选择里找满足条件的方案”的问题。
- 可以把它理解成在一棵“决策树”上搜索，每一层代表一次选择,一定要先画出决策树。
- 走到底得到一个结果；如果发现不合法，或者这一条路走不通，就退回上一层，换一个选择继续试。

### 必须掌握的 4 个核心点

- `路径 path`：当前已经做出的选择。
- `选择列表 choices`：当前这一步还能选什么。
- `结束条件`：什么时候把当前结果加入答案。
- `撤销操作`：递归返回前，把刚才加进去的状态恢复。

### 你要重点吃透的模板

- 回溯题大多都能套这个结构：

```java
void backtrack(参数) {
    if (满足结束条件) {
        收集答案;
        return;
    }

    for (每个可选项) {
        做选择;
        backtrack(下一层);
        撤销选择;
    }
}
```

- 真正的难点不是背模板，而是看清：
  1. 递归函数参数该传什么
  2. `for` 循环从哪里开始
  3. 哪些状态需要撤销
  4. 哪些情况要剪枝

### 学习时按题型分类

- `子集型`：每个元素选或不选。代表题：`78. 子集`
- `组合型`：从 n 个数里选 k 个。代表题：`77. 组合`
- `排列型`：元素顺序不同算不同答案。代表题：`46. 全排列`
- `去重型`：有重复元素，要避免重复结果。代表题：`40`、`47`、`90`
- `切割型`：按位置切字符串。代表题：`131. 分割回文串`
- `棋盘搜索型`：二维网格上搜索。代表题：`51. N 皇后`、`37. 解数独`

### Java 实现重点

- 路径通常用 `List<Integer> path = new ArrayList<>()`
- 结果通常用 `List<List<Integer>> res = new ArrayList<>()`
- 收集答案时要 `new ArrayList<>(path)`，不能直接放 `path`
- 撤销选择常见写法：

```java
path.add(nums[i]);
backtrack(...);
path.remove(path.size() - 1);
```

- 排列题常配合 `boolean[] used`
- 组合/子集题常用 `startIndex` 控制下一层从哪里开始，避免重复使用前面的元素

### 最容易犯错的地方

- 忘记复制路径，直接 `res.add(path)` 导致答案全错
- 递归返回后忘记撤销状态
- `startIndex` 和 `used[]` 混用不清
- 排序后不会写去重逻辑
- 结束条件写错，导致少结果或死递归

### 去重是回溯里的重点难点

- 先排序，再决定是否跳过重复元素
- 组合/子集常见去重：

```java
if (i > startIndex && nums[i] == nums[i - 1]) {
    continue;
}
```

- 排列去重要结合 `used[]`，这个是高频考点，必须单独练熟

### 剪枝要重点理解

- 剪枝就是“提前结束不可能成功的分支”，提升效率
- 例如组合题里，如果剩余元素数量不够，就不用继续递归
- 剪枝不是必须第一时间掌握，但做到中期一定要会，不然很多题会超时

### 最推荐的学习顺序

1. 先学决策树画法，理解“递归进去”和“回来撤销”
2. 刷 `77. 组合`
3. 刷 `78. 子集`
4. 刷 `46. 全排列`
5. 刷带重复元素的题：`90`、`40`、`47`
6. 再做 `131. 分割回文串`
7. 最后挑战 `51. N 皇后`

### 判断一道题是不是回溯

- 要求 `所有方案`
- 要求 `所有可能的组合/排列/切割方式`
- 每一步都有多个选择
- 需要试错，走不通再退回来

### 学习建议

- 不要一开始死记代码，先手画决策树
- 每做一道题都问自己 4 件事：`参数是什么`、`终止条件是什么`、`这一层做什么选择`、`怎么撤销`
- 一开始先不追求剪枝和最优写法，先把“能完整搜出来”做对
- 回溯的提升方式不是看懂，而是自己手写 5 到 10 题模板

### 一句话总结

- 回溯的核心不是“递归”，而是 `在搜索树上做选择、递归搜索、再撤销选择`。
