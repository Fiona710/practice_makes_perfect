# 问题

java程序员刷leetcode题目，对动态规划算法的原理以及实现都不了解，给出学习重点

# 回复

## 学习重点

- 先把动态规划理解成一句话：`把原问题拆成重复子问题，记录子问题答案，避免重复计算`
- 刷 `LeetCode` 时，不要先背题，先固定分析顺序：`状态` -> `转移方程` -> `初始化` -> `遍历顺序`
- 对 `Java` 程序员来说，真正难点通常不是语法，而是“怎么定义 `dp` 数组的含义”

## 先理解原理

- `动态规划` 适合两类特征：`重复子问题` 和 `最优子结构`
- `重复子问题`：同一个子问题会被反复计算，比如爬楼梯里 `f(n-1)`、`f(n-2)`
- `最优子结构`：大问题的最优解可以由小问题的最优解推出来
- 可以把它看成“带备忘录的递归”或“按顺序填表”

## 必须掌握的 4 个问题

- `dp[i]` 或 `dp[i][j]` 到底表示什么，这是第一关键
- 当前状态如何由之前状态推出，这是第二关键
- 初始值是什么，比如 `dp[0]`、`dp[1]`、第一行第一列
- 为什么要这样遍历，比如从前往后、从后往前、先外层物品再内层容量（从状态转移式中也可以看出遍历顺序，即dp[i]依赖的是dp[i-1]还是dp[i+1],需要把依赖项给计算出来，这样就能知道顺序是怎么来的了）

## 做题固定模板

- 第一步：定义状态  
  例子：`dp[i]` 表示“到第 `i` 个位置时的最优解”
- 第二步：写出转移方程  
  例子：`dp[i] = Math.max(dp[i - 1], dp[i - 2] + nums[i])`
- 第三步：确定初始化  
  例子：`dp[0] = ...`，`dp[1] = ...`
- 第四步：确定遍历顺序  
  因为 `dp[i]` 依赖前面的值，所以通常从小到大遍历
- 第五步：检查返回值  
  是返回 `dp[n]`、`dp[n - 1]`，还是整个表里的某个位置

## Java 实现重点

- 一维 `dp`：`int[] dp = new int[n + 1];`
- 二维 `dp`：`int[][] dp = new int[m + 1][n + 1];`
- 如果只依赖前几个状态，优先考虑`状态压缩`，减少空间
- 注意数组下标和题目下标是否一致，很多错误都出在这里
- `Java` 写法上常用 `Math.max()`、`Math.min()`，以及通过 `Arrays.fill()` 做初始化

## 最值得先学的题型

- `入门线性 dp`：爬楼梯、斐波那契、打家劫舍
- `路径类 dp`：不同路径、最小路径和
- `字符串 dp`：最长公共子序列、编辑距离、回文子串
- `背包 dp`：`0/1 背包`、完全背包、零钱兑换、目标和
- `区间 dp`：戳气球、最长回文子序列
- `股票 dp`：买卖股票系列，适合训练“状态定义”

## 初学者最容易犯的错

- 没想清楚 `dp` 含义就开始写代码
- 只会背公式，不知道公式为什么成立
- 初始化随便写，导致后面全错
- 遍历顺序错误，尤其是背包问题
- 把 `递归 + 记忆化` 和 `dp` 数组迭代当成完全不同东西，其实本质一致

## 推荐学习顺序

- 先学 `递归` 和 `记忆化搜索`，理解“子问题”
- 再做一维 `dp`
- 再做二维 `dp`
- 再学背包问题
- 最后学区间 `dp`、状态机 `dp`、树形 `dp`

## 刷题时的自问清单

- 这道题能不能拆成更小规模的同类问题？
- 子问题答案会不会重复使用？
- 我定义的状态能不能完整描述问题？
- 当前状态依赖哪些更小状态？
- 初始条件是什么？
- 遍历方向是否满足依赖关系？

## 学习建议

- 每道题先手写“状态定义”和“转移方程”，再写代码
- 一开始不要追求做很多题，先把同一题用 `递归`、`记忆化`、`dp` 三种方式各写一遍
- 做完题后复盘：这题属于哪一类、状态为什么这样定义、能不能压缩空间
