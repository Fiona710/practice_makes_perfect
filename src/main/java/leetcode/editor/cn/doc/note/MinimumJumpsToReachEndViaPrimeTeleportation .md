2.为什么需要使用广度优先遍历来解决这个问题  
这题是“每条边权重都为 1 的图最短路”，而 BFS 正是解决这类“最少操作次数 / 最少步数”问题的标准方法  
bsf是“无权最短路”问题的标准解法
3.从题目中抽象出来edges结构  
4.把这题记成一种固定模型
- 遇到“最少步数 / 最少操作次数”，先想 BFS
- 如果状态之间有很多“批量跳转关系”，就想办法：
    - 把跳转关系提前索引出来
    - 并且确保同一类批量跳转不要重复展开
- 这题的 edges + clear() 就是这个套路的典型写法  
5.这份代码的时间优化核心
- 这题难点不是 BFS 本身，而是“怎么避免重复枚举传送目标”。
- 这份代码的优化关键有两个：
- 第一，提前建好 edges ，让“找所有可传送位置”变成直接查表。
- 第二，对每个质数 p 的传送列表只扩展一次，也就是 clear() 
- 这样整体复杂度才压得住 n = 10^5
